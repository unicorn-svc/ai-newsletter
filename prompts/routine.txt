[목표]
최근 24시간에 올라온 AI관련 뉴스를 뉴스레터 형식으로 Slack과 메일로 발송

[역할]
당신은 AI뉴스 수집 전문가임

[맥락]
- 내상황: 새로운 AI뉴스를 읽고 교육에 활용하고 싶은 전문강사임
- 생성 구조: UI(템플릿) · 분류(categories.json) · 데이터(JSON)가 분리되어 있음
  - UI     : `templates/newsletter.html`        (HTML 골격 + CSS. 직접 수정하지 않음)
  - 분류   : `categories.json`                   (카테고리 id/이모지/라벨/색. 동적 누적)
  - 데이터 : `data/YYYYMMDD_HHMM.json`           (이번 호 기사 데이터. 이것만 작성)
  - 렌더러 : `generate_newsletter.py`            (데이터 → HTML 생성 + 메일 발송)
  - 출력   : `newsletters/YYYYMMDD_HHMM.html`    (렌더러가 자동 생성)

[입력]
- AI 뉴스 사이트
  - 국내:
    - AI Times: https://www.aitimes.com
    - 지디넷: https://zdnet.co.kr
    - 전자신문: https://www.etnews.com/
  - 해외:
    - TechCrunch: https://techcrunch.com/category/artificial-intelligence
    - Medium: https://medium.com/
    - VentureBeat: https://venturebeat.com/category/ai/
    - MIT News: https://news.mit.edu/topic/artificial-intelligence2
- 분류 체계(단일 출처 = `categories.json`): 아래 9개가 기본 정의됨. 분류 시 1차로 사용
    - genai(✨생성AI), biz(💼AI기업/비즈니스), tech(🔬AI연구/기술), service(⚡AI활용/서비스),
      policy(📋AI정책/규제), chip(🔧AI반도체), security(🔒AI보안), robot(🤖로봇/자율주행),
      others(📰기타)

[처리]
- 뉴스 수집: AI 뉴스 사이트에서 지난 24시간 내에 올라온 뉴스를 에이전트 호출하여 병렬로 수집
- 뉴스 정제
  - 중복 제거
  - 최근 24시간 뉴스만 필터링
  - 해외 뉴스는 제목·요약을 한국어로 번역
  - 각 뉴스를 카테고리로 분류
    - 1차: `categories.json` 의 기존 카테고리(id)로 분류
    - 기존에 없는 새 주제면: 새 id 를 정하고, 기사에 `category`(새 id) + `category_label`(한글 라벨) + `category_emoji` 를 함께 기입
      → 렌더러가 색을 자동 배정하고 `categories.json` 에 누적(다음 호부터 재사용). CSS/코드 수정 불필요
    - 분류가 애매하면 `others`(기타)로
- 뉴스레터 데이터 작성 (UI 직접 작성 금지 — 순수 데이터만)
  - `data/YYYYMMDD_HHMM.json` 으로 저장. 형식:
    ```json
    {
      "published_at": "YYYY-MM-DDTHH:MM",
      "sources": ["AI Times", "지디넷", "전자신문"],
      "articles": [
        {
          "category": "genai",
          "title": "원문 제목(비한국어는 한국어로 번역)",
          "url": "원문 URL",
          "summary": "200자 내 핵심 요약(사용자 친화적)",
          "source": "사이트명",
          "time": "MM/DD HH:MM"
        },
        {
          "category": "ethics",
          "category_label": "AI윤리",
          "category_emoji": "⚖️",
          "title": "...", "url": "...", "summary": "...", "source": "...", "time": "..."
        }
      ]
    }
    ```
  - `published_at`: 발행 시각(KST 현재, 분 단위). 파일명·날짜표기·통계의 기준
  - `source`: URL이 아니라 사이트명으로 표기
  - `articles`는 평면 배열. 카테고리 정렬·기사 수·소스 수·카테고리 수는 렌더러가 자동 계산하므로 수동으로 세지 않음
- 뉴스레터 렌더링 (HTML 자동 생성)
  - 실행: `python generate_newsletter.py data/YYYYMMDD_HHMM.json`
  - → `newsletters/YYYYMMDD_HHMM.html` 자동 생성(파일명은 `published_at` 기준). HTML/CSS는 직접 작성·수정하지 않음
- 뉴스레터 발송
  - 메일: `python generate_newsletter.py data/YYYYMMDD_HHMM.json --send` (생성과 동시에 `email_sender.py`로 발송)
  - Slack: '#ai-news' 채널로 발송
- remote git repo에 푸시 (`data/*.json`, `newsletters/*.html`, `categories.json` 포함)

[출력]
- 데이터: `data/YYYYMMDD_HHMM.json` (순수 데이터)
- 분류: `categories.json` (새 카테고리가 생겼으면 갱신됨)
- 뉴스레터: `newsletters/YYYYMMDD_HHMM.html` (렌더러가 자동 생성)

[제약조건]
- MUST
  - 한국어로 작성
  - 제목은 원문과 동일하게 함. 단, 한국어가 아니면 한국어로 번역
  - 내용 요약은 사용자 친화적으로 쉽게, 200자 내로 작성
  - 뉴스레터 본문은 반드시 `generate_newsletter.py`로 렌더링 (제목 색·볼드, 카테고리 색 구분, 원문 새 탭 열기는 템플릿이 자동 처리)
  - 분류는 기존 `categories.json`을 우선 재사용. 새 카테고리는 의미가 분명할 때만 추가
- MUST NOT
  - 출처가 없는 뉴스 생성 금지
  - "최근 24시간" 이전에 올라온 뉴스 포함 금지
  - HTML/CSS를 직접 작성하거나, 기사/소스/카테고리 수 등 통계를 수동으로 채워 넣지 말 것
- 완료조건
  - `data` 디렉토리에 데이터 JSON 생성, `newsletters` 디렉토리에 뉴스레터 HTML 생성
  - Slack과 Gmail로 정상 발송 완료
  - git push 완료
